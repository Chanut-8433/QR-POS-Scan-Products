QR Price POS Offline (Browser Fix)

ไฟล์ในชุดนี้:
- index.html
- app.js
- service-worker.js
- manifest.json
- qrgen.js
- icon-192.png
- icon-512.png
- README.txt

สิ่งที่แก้ไขรอบนี้:
- แก้ปัญหา "เบราว์เซอร์ไม่รองรับ BarcodeDetector"
- เปลี่ยนโหมดสแกนจาก BarcodeDetector ตรงๆ ไปใช้ไลบรารี qr-scanner (legacy build)
- ไลบรารีนี้มี JavaScript fallback ทำให้รองรับเบราว์เซอร์ได้กว้างขึ้น
- Service worker จะพยายาม cache ไฟล์สแกนจาก CDN เอาไว้ เพื่อให้เปิดใช้ออฟไลน์ได้หลังเปิดครั้งแรก

หมายเหตุสำคัญ:
- ครั้งแรกต้องเปิดออนไลน์ 1 ครั้ง เพื่อให้เว็บโหลดและ cache ไฟล์สแกนจาก jsDelivr
- หลังจากนั้นจึงค่อยใช้ออฟไลน์ได้
- ถ้าครั้งแรกเปิดแล้วอินเทอร์เน็ตหลุดก่อนโหลดสคริปต์สแกนสำเร็จ โหมดสแกนอาจยังไม่พร้อม ให้เปิดออนไลน์อีกครั้งแล้วเข้าแอปอีกครั้ง

วิธีใช้งาน:
1) อัปโหลดทั้งโฟลเดอร์นี้ขึ้นโฮสต์ HTTPS
2) เปิดด้วยเบราว์เซอร์ตอนมีเน็ต 1 ครั้ง
3) รอจนหน้าโหลดครบ แล้วทดลองกดเปิดกล้องสแกน 1 ครั้ง
4) ติดตั้งเป็น PWA (Add to Home Screen)
5) หลังจากนั้นเปิดจากไอคอนแอปเพื่อใช้งานออฟไลน์

รูปแบบ QR ที่รองรับ:
- 25
- ชื่อสินค้า|25
- SKU|ชื่อสินค้า|25

โหมดสร้าง QR:
- กรอกรหัสสินค้า (ไม่บังคับ)
- กรอกชื่อสินค้า
- กรอกราคา
- กดสร้าง QR
- บันทึกภาพ PNG หรือแชร์ภาพได้

ความเข้ากันได้:
- Android Chrome/Edge: เหมาะมาก
- iPhone Safari / iOS PWA: ดีกว่า BarcodeDetector ตรงๆ แต่ยังขึ้นกับสิทธิ์กล้องและเวอร์ชันของระบบ
- ต้องใช้ HTTPS / localhost / PWA สำหรับกล้อง
